# Task Execution Standard

## Basic Flow

1. Owner gives a request.
2. Studio Director identifies the real task.
3. Director decides whether a role call is needed.
4. Context package is kept small.
5. Selected role answers only the assigned task.
6. Director summarizes the result.
7. Owner approval is requested when needed.
8. Accepted result is stored.

## Task Shape

Every task should have:

- title
- goal
- project
- role needed
- context needed
- expected output
- completion rule

## Output Shape

Default output:

- Result
- Risk
- Missing information
- Next action

## Completion Rule

A task is complete only when its expected output exists and the Director marks it ready for owner review.

## Final Principle

Small tasks move faster than large unclear tasks.
