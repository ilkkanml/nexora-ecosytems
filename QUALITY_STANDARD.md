# Quality Standard

NEXORA Studio OS separates checked information from assumptions.

Core rules:

- Use inspected project data when available.
- Label assumptions clearly.
- Label missing information clearly.
- Keep answers within the assigned task.
- Avoid unrelated suggestions.
- Use completion rules before marking work complete.
- Keep department outputs compact.

Evidence examples:

- file inspection
- command output
- test output
- API output
- owner-approved decision

Final principle:

Accuracy comes before speed.
