# Project Intake

## Purpose

This document defines how a future project enters NEXORA Studio OS.

No project starts until it has a clear intake record.

## Required Project Intake Fields

Every project must define:

- name
- type
- purpose
- owner goal
- target users
- repository plan
- hosting needs
- data needs
- risk level
- first milestone
- success definition

## Project Types

Supported types:

- WEB_DESIGN
- WEB_APP
- MOBILE_APP
- TV_APP
- GAME
- BACKEND_PLATFORM
- INTERNAL_TOOL

## Required Foundation Documents Per Project

A project should have:

- project constitution
- roadmap
- milestone list
- task list
- role plan
- data rules
- decision log

## Project Start Rule

A project may start only after:

- Studio Director reviews the intake
- Product Architect defines the scope
- Builder identifies technical shape
- QA Reviewer identifies risk
- Owner approves the project start

## Minimal Project Workflow

1. Create intake.
2. Create constitution.
3. Create roadmap.
4. Create first milestone.
5. Create first task set.
6. Approve implementation target.
7. Start work.

## Final Rule

A project without scope, milestone, and approval stays in draft.
